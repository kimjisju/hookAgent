# 로드맵

## 다음 구현 계획

### 1. 실시간 GUI 기반 추적

구현 완료

현재처럼 단순히 로그로 남기는 것을 넘어, Claude 실행 시 새로운 GUI를 띄워 다음 정보를 실시간으로 추적할 수 있게 합니다.

- 사용자 입력
- 추론
- 도구 사용

목표는 로그 파일을 나중에 읽는 방식이 아니라, 에이전트의 현재 상태와 흐름을 실시간으로 관찰할 수 있게 하는 것입니다.

### 2. 도구 사용 전 중단 및 사용자 승인 제어

구현 완료

단순 추적을 넘어서, 도구 사용 전에 에이전트를 멈추고 GUI 앱에서 사용자의 판단(`Yes`, `No`)으로 도구 사용 여부를 제어할 수 있게 합니다.

이 단계의 핵심은 다음과 같습니다.

- 도구 실행 직전 인터셉트
- 에이전트 일시 중지
- 사용자 승인 또는 거부
- 사용자 판단에 따라 재개 또는 차단

### 3. Claude Code 외 다른 AI Agent 지원

대상 범위를 Claude Code에 한정하지 않고 다음과 같은 다른 AI Agent까지 확장합니다.

- gemini-cli
- Codex
- open claw
- 그 외 유사한 Agent 환경

확장 목표는 각 에이전트의 사용자 입력, 추론, 도구 사용을 가로채고, 도구 사용 전 에이전트를 중지시킨 뒤 사용자의 판단에 따라 진행 여부를 제어하는 공통 계층을 만드는 것입니다.

### 4. 위험 감지 모델을 통한 자동 제어

최종적으로는 사용자의 수동 판단만으로 제어하는 것을 넘어서, 위험 감지 모델이 다음 4가지 과정을 자동으로 수행해 제어하도록 합니다.

1. 사용자 입력에 프롬프트 인젝션 내용이 없는가?
2. 사용자 입력에 따른 정당한 추론인가?
3. 사용자 입력에 따른 추론에 따른 정당한 도구 사용인가?
4. 도구 사용 결과가 추론에 부합하는가?

이 단계에서는 사람이 매번 판단하지 않아도, 시스템이 위험도를 자동으로 평가하고 허용, 보류, 차단 같은 제어 결정을 내리는 구조를 목표로 합니다.

## 단계별 진화 방향

프로젝트는 다음 흐름으로 진화합니다.

1. 로그 기반 추적
2. 실시간 GUI 관찰
3. 사용자 승인 기반 제어
4. 다중 Agent 지원
5. 위험 감지 모델 기반 자동 제어

현재 위치는 3단계 초입이며, Claude Code 기준으로는 실시간 GUI 관찰과 사용자 승인 기반 제어까지 구현되었습니다. 이후 단계는 이 구조를 다중 Agent와 자동 위험 판단으로 확장하는 방향입니다.
